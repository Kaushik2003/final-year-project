\chapter{Work Plan}
\label{Work_Plan}

\section{Overview}
The project spans twelve months across two semesters (Project-I: Months 1--5; Project-II: Months 6--12) and is divided into seven phases. Each phase produces a verifiable deliverable, ensuring early hardware validation and incremental software integration, with field evaluation beginning after core system validation.

\section{Project-I (Months 1--5)}

\subsection{Phase 1: Drone Build and Sensor Validation (Months 1--3)}

\textbf{Activities:}
\begin{itemize}
    \item Procure hardware (motors, ESCs, Pixhawk, battery, LD2410, GPS, Pi Camera).
    \item Assemble F450 frame and power system; configure Pixhawk (ArduPilot).
    \item Calibrate ESCs, accelerometer, and compass.
    \item Validate sensors: camera (Picamera2), radar (UART 256000 baud), GPS.
    \item Mount sensors using 3D-printed fixtures.
    \item Perform tethered hover tests and GPS position hold.
    \item Complete DGCA registration if required.
\end{itemize}

\textbf{Deliverable:} Airworthy drone with validated sensors and stable hover.

\subsection{Phase 2: Edge AI and Radar Processing (Months 4--5)}

\textbf{Activities:}
\begin{itemize}
    \item Deploy YOLOv8-pose (INT8, ONNX) on RPi~4 ($\geq$3~fps).
    \item Integrate ByteTrack for tracking.
    \item Implement CFAR/MTI/Doppler radar pipeline using LD2410 data.
    \item Develop multi-threaded pipeline (camera, radar, inference, GPS).
    \item Design MQTT schema and test streaming to backend.
\end{itemize}

\textbf{Deliverable:} Working edge AI + radar pipeline with verified MQTT stream and performance metrics.

\section{Project-II (Months 6--12)}

\subsection{Phase 3: Backend and Behavior Analysis (Months 6--7)}

\textbf{Activities:}
\begin{itemize}
    \item Develop FastAPI/MQTT backend and WebSocket streaming.
    \item Implement SQL database and behavior rules (loitering, breach, etc.).
    \item Add Kalman trajectory prediction (5~s ahead).
    \item Test live drone-to-backend streaming and alarms.
\end{itemize}

\textbf{Deliverable:} End-to-end system with behavior detection and database logging.

\subsection{Phase 4: Fusion MLP and Evaluation (Months 7--8)}

\textbf{Activities:}
\begin{itemize}
    \item Collect $\sim$500 field samples (positive/negative scenarios).
    \item Train six-feature MLP (Adam, BCE loss, 80/20 split).
    \item Evaluate vision-only, radar-only, and fused models (precision, recall, $F_1$).
    \item Analyse robustness under degraded conditions.
\end{itemize}

\textbf{Deliverable:} Trained ONNX MLP and benchmark evaluation report.

\subsection{Phase 5: War-Room Dashboard (Months 8--9)}

\textbf{Activities:}
\begin{itemize}
    \item Extend NASA Worldview (React + OpenLayers).
    \item Add telemetry layers, detection markers, trajectories, MJPEG feed.
    \item Implement alarm panel and LLM query interface.
    \item Integrate external layers (OpenSky, CelesTrak).
\end{itemize}

\textbf{Deliverable:} Functional dashboard with real-time visualization and LLM queries.

\subsection{Phase 6: Soldier Mobile Application (Months 10--11)}

\textbf{Activities:}
\begin{itemize}
    \item Develop React Native + ARCore app.
    \item Implement AR overlays, mini-map, compass, and TTS alerts.
    \item Test multi-device real-time synchronization.
\end{itemize}

\textbf{Deliverable:} Working AR-based mobile application.

\subsection{Phase 7: Integration and Final Evaluation (Month 12)}

\textbf{Activities:}
\begin{itemize}
    \item Perform full system integration and field trials.
    \item Evaluate metrics (latency, $F_1$, tracking, robustness).
    \item Prepare report, demo video, and research paper.
    \item Release open-source code, dataset, and models.
\end{itemize}

\textbf{Deliverable:} Fully integrated prototype, evaluation results, and open-source release.
